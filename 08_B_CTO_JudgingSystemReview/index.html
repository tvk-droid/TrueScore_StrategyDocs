CONSTRAINTS ARCHITECTURE DRIVERS
ARCHITECTURE DRIVERS

These Four Constraints Drive Every Design Decision

LATENCY
<200ms

Real-time judge assistance requires end-to-end latency under 200ms. Camera ‚Üí AI ‚Üí tablet must complete within one human reaction time.

EXPLAINABILITY
Level 4

Every AI decision must be explainable with video evidence. Skeleton overlay, angle measurements, clip extraction. "Because the AI said so" is not acceptable.

GLOBAL SHUTTER
0.5ms

Tumbling athletes move at 1000¬∞/sec. Rolling shutter cameras introduce 15-25% accuracy loss. Global shutter (0.5ms exposure) is non-negotiable.

SCORING WINDOW
60-90s

Routine ends ‚Üí score required in 60-90 seconds. All processing must complete within this window or judges fall behind.

Constraints derived from competition operations research, camera hardware analysis, judge workflow interviews

# Standing Back Tuck - Deduction Engine
class StandingBackTuck:
    def __init__(self):
        self.phases = ["PREP", "TAKEOFF", "FLIGHT", "LANDING"]
        self.flight_rotation_required = 360  # degrees
        self.knee_tuck_threshold = 45  # degrees
        
    def detect_deductions(self, skeleton_sequence):
        deductions = []
        
        # Check rotation completion
        hip_rotation = self.calc_hip_rotation(skeleton_sequence)
        if hip_rotation < 340:  # 20¬∞ tolerance
            deductions.append({
                "code": "INCOMPLETE_ROTATION",
                "severity": 0.3,
                "evidence": self.get_rotation_clip()
            })
        
        # Check body position
        knee_angle = self.calc_knee_angle(skeleton_sequence, "FLIGHT")
        if knee_angle > self.knee_tuck_threshold:
            deductions.append({
                "code": "LOOSE_TUCK",
                "severity": 0.1,
                "evidence": self.get_angle_overlay()
            })
            
        return deductions
